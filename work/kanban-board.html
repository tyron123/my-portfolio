Kanban-board Redesign

  • Role: Senior UX Designer
  • Company: TOPdesk GmbH
  • Duration: ~9 months
  • Team: 1 PO, 4 developers, 1 tester, 1 designer, 1 scrum master, 1 solutions architect
  • Tools: Figma, Useberry, Miro, Jira
  • Outcomes: 23% reduction in task completion time; 9/10 user satisfaction; 2+ design system components contributed

1. The Call to Adventure

The Kanban board wasn't broken — but it wasn’t whole either. Initially, the project aimed to integrate a long-requested Operations Management (OM) module alongside the existing Incidents and Changes already displayed...

2. The Challenge

Constraints: Time pressure, legacy tech, and new insights on OM behavior...

Accessibility: Introduced tab order and keyboard shortcuts to seed inclusive design.

3. The Journey

Research: Interviews, usage data review, prototype tests (task setup, closing, filters, pagination).

Key insights: Filters mattered most; >50 tasks per column desired but backend-limited.

Iteration: 10+ rounds across closure flows, OM tasks, filter placement, pagination, setup wizards.

Design System: New side panel layout, updated card UI, wizard-style stepper.

4. The Transformation

  • Full UI refresh via design system
  • Filter bar for task types
  • Keyboard accessibility
  • Toggle for closed tasks
  • Seamless OM integration

Outcomes: 23% faster task completion; 9/10 satisfaction.

5. The Return

What I learned: Technical limits can block good intent; prioritization needs the whole team; working ahead enables polish.

Next: Setup wizard redesign, advanced filters, stronger a11y, real pagination/infinite scroll.

Crafting seamless, empowering experiences that connect people, purpose, and technology.
